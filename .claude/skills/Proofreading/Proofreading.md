# Proofreading Skill

## Purpose
The Proofreading skill ensures content quality by checking for grammar, style consistency, and compliance with project specifications. It validates that all content meets quality standards before publication.

## Capabilities
- Grammar and spelling validation
- Style consistency checking
- Specification compliance verification
- Frontmatter validation
- Content structure validation

## Validation Checks

### 1. Grammar and Language
- Check for grammatical errors
- Verify spelling and punctuation
- Ensure proper sentence structure
- Validate subject-verb agreement

### 2. Style Consistency
- Verify consistent terminology usage
- Check for consistent formatting
- Ensure voice and tone consistency
- Validate citation and reference style

### 3. Specification Compliance
- Verify content aligns with book specification
- Check that all required sections are present
- Validate that content meets defined objectives
- Ensure compliance with project requirements

### 4. Frontmatter Validation
- Verify required frontmatter fields are present
- Check for proper YAML formatting
- Validate field values and data types
- Ensure consistency across documents

### 5. Structure Validation
- Verify proper document structure
- Check for appropriate headings hierarchy
- Validate cross-references and links
- Ensure proper content flow

## Example Triggers
- Content creation: Automatically validate new content
- Content update: Re-validate updated sections
- Quality gate: Block publication of low-quality content
- Specification change: Re-validate affected content

## Guardrails
- Prevents publication of content with grammar errors
- Ensures all content meets quality thresholds
- Blocks content that doesn't comply with specifications
- Maintains consistent style across the entire book
